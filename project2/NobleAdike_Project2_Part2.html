Project 2 Part 1: Exploratory Data Analysis¶

1. Dataset Information¶

Dataset Title and Topic¶

Sports Data Analysis: Analysing official FIFA player statistics and rankings data.

Data Source¶

The dataset is sourced from the FIFA player statistics CSV file (fifa_eda_stats.csv) on a Kaggle project, containing player performance metrics and rankings information.

Real-World Context and Purpose¶

This dataset represents international football player statistics and rankings, which is valuable for:

  • Understanding player performance metrics across different categories
  • Analyzing ranking trends and player evaluations
  • Supporting sports analytics and player comparison studies

Dataset Introduction¶

This dataset contains statistics for 15,000+ FIFA players, sourced from FIFA player ratings data (fifa_eda_stats.csv). It includes 57 columns covering player identity (name, age, nationality, club), financial information (market value, wage, release clause), physical attributes (height, weight), and over 30 in-game skill ratings such as dribbling, passing, shooting, and goalkeeping.

Data Type Conversion¶

Several columns need their data types converted before analysis. Value, Wage, and Release Clause are currently stored as strings because of the currency symbol and suffixes (e.g. "€110.5M"), so the symbol and letters need to be stripped and the values converted to floats. Similarly, Height is stored as a string in feet and inches format (e.g. "5'7"") and Weight has a "lbs" suffix so both need to be converted to plain numbers so they can be used for numerical analysis. Joined and Contract Valid Until are date strings that should be converted to datetime so they can be used in time-based analysis.

Columns with unusual or inconsistent values:

Data Quality Observations¶

Looking at the missing values, most columns are in good shape. The skill stat columns (Crossing, Dribbling, etc.) along with Height, Weight, Preferred Foot and others all have the same 48 rows missing. These are incomplete player profiles that we will need to drop entirely. Club has 241 missing values which just means those players are free agents, so we fill those with "Free Agent". Loaned From has the most missing values at 16,943 (93%) but this is expected as it only applies to players currently on loan, so the rest can be filled with "Not on Loan". Position and Jersey Number have 60 missing values each and we can fill with "Unknown" and the median respectively. Joined, Contract Valid Until, and Release Clause might need to be left missing or dropped entirely since these cannot be reliably guessed or imputed. No duplicate rows were found in the dataset.

Strategy to handle missing data for dataset¶

  • 48 rows missing all skill stats + Height/Weight/Preferred Foot so dropping these rows entirely (incomplete profiles, not salvageable)
  • Club column (241) — fill with "Free Agent"
  • Loaned From (16,943) — fill with "Not on Loan" (it's structural, not missing data)
  • Joined / Contract Valid Until / Release Clause — leave as NaN (can't reliably impute contract dates/clauses)

def parse_currency(val):
    if pd.isna(val):
        return np.nan
    val = str(val).replace('€', '').strip()
    if 'M' in val:
        return float(val.replace('M', '')) * 1_000_000
    elif 'K' in val:
        return float(val.replace('K', '')) * 1_000
    else:
        return float(val)

def parse_height(h):
    if pd.isna(h):
        return np.nan
    parts = str(h).replace('"', '').split("'")
    feet = int(parts[0])
    inches = int(parts[1]) if len(parts) > 1 and parts[1] else 0
    return round((feet * 12 + inches) * 2.54, 2)

def parse_weight(w):
    if pd.isna(w):
        return np.nan
    return float(str(w).replace('lbs', ''))

10. Initial Observations¶

This dataset captures over 18,000 male professional football players across 160+ nationalities, and several patterns emerged during exploration. The most striking was the number of young players (23 and under) with a potential rating of 85 or above (over 50 prospects) suggesting the sport is in a strong era for emerging talent. The GK stat columns showed a clear bimodal pattern, where outfield players score near zero and only goalkeepers rate highly, making these columns misleading if analysed without separating positions. There was also a noticeable gap between physical attributes like Strength and Jumping (averaging above 60) and technical ones like Free Kick Accuracy and Volleys (averaging in the low 40s). The grouped data confirmed that popular positions like strikers tend to have lower average Overall ratings than rarer roles, and that England leads in player count but not in average quality. One small curiosity was the Body Type column, where a handful of players have their own name as their body type; it suggested FIFA's way of flagging physically unique players.

On the data quality side, the dataset required meaningful cleaning before it could be used analytically. The same 48 rows were missing values across all skill stats, height, weight, and preferred foot — these were dropped entirely as unrecoverable. Financial columns like Value, Wage, and Release Clause were stored as currency strings and needed parsing into floats, while Height and Weight required unit conversion to be numerically useful. Free agents were identified in the Club column and labelled accordingly, and the Loaned From column (missing for 93% of players) was filled to reflect that most players are simply not on loan. After cleaning, a BMI column was derived from the converted height and weight values to add a standardised measure of player physicality.

SUMMARY¶

In this project, we analyzed a FIFA player dataset containing statistics for 18,000+ professional football players across 57 columns. The first phase focused on data exploration and cleaning. We looked at the dataset's structure, data types, descriptive statistics, and missing values, then made a plan to handle each issue. The 48 rows missing all skill stats were dropped entirely, free agents were labeled in the Club column, loan status was filled in for the majority of players not on loan, and smaller gaps in Position and Jersey Number were handled with "Unknown" and the median respectively. We also converted currency columns like Value, Wage, and Release Clause from string format into usable floats, converted Height and Weight into standard units, and parsed date columns into datetime. A BMI column was then derived from the cleaned height and weight values as part of the data wrangling phase.

The second phase moved into visualization and grouped analysis. We used barplots, line plots, a pie chart, and a heatmap to explore patterns in the data. The country barplot showed that Brazil, Spain, and France produce the most elite players despite England having the largest raw player count. The age vs overall line plot revealed a peak around age 31 driven by survivorship bias. The age vs market value line plot showed that value decline usually starts at around age 27.The preferred foot pie chart confirmed the roughly 75/25 right-to-left split seen in real-world football. The heatmap showed that most technical skills are strongly correlated with each other, with HeadingAccuracy being the clear outlier due to its physical rather than technical nature.
